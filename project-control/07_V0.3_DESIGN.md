# StudyQuest V0.3 AI 出题设计

## 目标

使用本机运行的 Ollama 模型生成结构化题目草稿，在 Streamlit 中预览，
经用户确认后保存到现有 SQLite 题库。保存过程需要避免重复题目，并且不产生
按请求计费的 API 费用。

## 模型服务选择

使用 Ollama 官方 Python 客户端及其原生结构化输出能力。默认模型为
`qwen3:8b`，该模型支持中文，下载体积约为 5.2 GB。用户可以在 `.env`
中修改模型名称和服务地址。

Ollama 作为独立的本地后台服务运行。StudyQuest 不自动安装 Ollama，也不自动
下载模型，避免用户在网页中点击按钮后意外触发大文件下载或占用大量磁盘空间。

## 配置

StudyQuest 使用 `python-dotenv` 从项目根目录加载 `.env`：

```dotenv
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen3:8b
```

本方案不需要 API Key。Ollama 不可用时，页面仍可正常打开，并显示明确的安装、
启动或模型下载命令，而不是直接报错退出。

## 模块划分

- `modules/ai_generator.py`：负责配置、Pydantic 响应模型、提示词构造、
  Ollama 调用、结果标准化和业务校验。
- `modules/importer.py`：提供 CSV 导入和 AI 题目保存共用的去重保存函数。
- `app.py`：提供 AI 出题表单、运行状态提示、会话预览和保存操作。

## 生成流程

1. 加载 `.env`，显示主题、题型、数量（1 至 20）和难度控件。
2. 用户点击 Generate Questions 后，使用 `QuestionBatch` JSON Schema 和
   `temperature=0` 调用一次 Ollama。
3. 按必填字段和题型规则校验每道题。
4. 将判断题答案统一为 `True` 或 `False`，将多选题答案统一为顺序稳定的
   英文逗号分隔标签，并将非选择题的选项统一为空列表。
5. 将校验通过的题目保存到
   `st.session_state.ai_generated_questions`。
6. 在写入 SQLite 前预览全部必需字段。
7. 用户点击 Save to Question Bank 后，通过共用的去重保存函数写入，
   `source` 固定为 `ai`。
8. 显示生成数量、新增数量、跳过重复数量和当前题库总数。

如果模型返回的有效题目少于请求数量，保留已经通过校验的题目并显示数量不足
提示。V0.3 不自动发起第二次补充生成请求。

## 校验规则

- 每道题的 `subject`、`type`、`stem`、`answer`、`explanation`、
  `tags` 和 `difficulty` 均不能为空。
- `type` 必须与用户请求的题型一致，并属于四种受支持题型之一。
- `difficulty` 必须与用户请求的难度一致。
- 单选题至少包含两个带标签的选项，答案必须对应其中一个有效标签。
- 多选题至少包含两个带标签的选项，答案包含一个或多个不重复的有效标签；
  批改时不考虑标签顺序。
- 判断题和简答题的 `options` 统一保存为 `[]`。
- 判断题支持的中英文答案会统一转换为 `True` 或 `False`。
- 响应根对象必须包含列表类型的 `questions` 字段。

## 错误处理

- Ollama 未安装或服务无法连接：显示安装和启动说明。
- 配置的模型不存在：显示 `ollama pull <model>` 命令。
- 请求超时或传输失败：显示简洁错误，并保留此前已有的预览结果。
- 返回内容不符合 JSON Schema：显示响应校验错误。
- 业务字段缺失或无效：指出具体出错的题目。
- 返回数量不足：显示警告并保留有效题目。
- 保存时发现重复题：跳过并计入保存统计。

## 测试范围

- 正常解析有效的结构化响应。
- 拒绝非 JSON、字段缺失、选项无效或答案不兼容的响应。
- AI 题目以 `source = ai` 保存。
- 重复保存同一道 AI 题目时不会新增记录。
- Ollama 服务不可用时页面不会崩溃。
- 保存后的 AI 题目可以被现有 Quiz 查询抽取。
- 保持全部 V0.2 测试通过。

## 本地配置步骤

1. 安装 Windows 版 Ollama。
2. 首次执行 `ollama pull qwen3:8b` 下载模型。
3. 确认 Ollama 正在运行，默认 API 地址为
   `http://localhost:11434`。
4. 正常启动 StudyQuest，不需要配置密钥或购买付费账户。

## 本轮不做

本轮不自动安装 Ollama、不自动下载模型、不提供云模型自动回退、不进行 AI 简答题
批改、不自动补生成题目、不保存提示词历史、不制作使用量图表，也不开发 PDF、
Word、OCR、登录注册、复杂界面及任何 V0.4 功能。
