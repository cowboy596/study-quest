# StudyQuest V0.3 AI Generate Design

## Goal

Use the OpenAI Responses API to generate strictly structured question drafts,
preview them in Streamlit, and save approved drafts into the existing SQLite
question bank without duplicates.

## API Choice

Use the official OpenAI Python SDK and `client.responses.parse(...)` with Pydantic
Structured Outputs. The default model is `gpt-5.4-mini`; users may override it in
`.env`. `AI_API_BASE_URL` is optional and must refer to a Responses-compatible API.

## Configuration

StudyQuest loads `.env` from the project root with `python-dotenv`:

```dotenv
AI_API_KEY=replace_with_your_secret_key
AI_API_BASE_URL=https://api.openai.com/v1
AI_MODEL=gpt-5.4-mini
```

`AI_API_KEY` is required for generation. The base URL and model use the values
above when omitted. The page renders its inputs when the key is missing, shows a
warning, and disables generation instead of raising an exception. `.env` remains
ignored by Git; only `.env.example` is committed.

## Components

- `modules/ai_generator.py`: configuration, Pydantic response schema, prompt
  construction, OpenAI client call, normalization, and domain validation.
- `modules/importer.py`: shared duplicate-safe save function used by CSV and AI.
- `app.py`: AI form, session-state preview, warning/error messages, and save action.

## Generation Flow

1. Load `.env` and render topic, type, count (1-20), and difficulty controls.
2. On Generate, call the Responses API once with a strict `QuestionBatch` schema.
3. Validate every question against required fields and type-specific rules.
4. Normalize true/false answers to `True` or `False`, multiple-choice answers to a
   stable comma-separated label list, and non-choice options to an empty list.
5. Store validated dictionaries in `st.session_state.ai_generated_questions`.
6. Preview all required fields without writing to SQLite.
7. On Save, use the shared insert-or-ignore path with `source = ai`.
8. Show generated, inserted, duplicate, and current total counts.

If the model returns fewer valid questions than requested, keep the valid preview
and show an explicit shortfall warning. V0.3 does not make a second API call.

## Validation Rules

- Every item requires non-empty `subject`, `type`, `stem`, `answer`,
  `explanation`, `tags`, and `difficulty`.
- Type must equal the requested type and be one of the four supported types.
- Difficulty must equal the requested difficulty.
- Single choice requires at least two labeled options and one valid answer label.
- Multiple choice requires at least two labeled options and one or more valid,
  unique answer labels; order does not affect grading.
- True/false and short-answer questions store `options` as `[]`.
- True/false answers normalize supported English or Chinese values.
- The response root must contain `questions` as a list.

## Error Handling

- Missing key: warning and disabled Generate button.
- API authentication, rate-limit, timeout, or network failure: concise page error;
  no preview is replaced with partial raw data.
- Refusal, absent parsed output, or non-JSON/schema output: response error.
- Missing or invalid domain fields: validation error identifying the item.
- Count shortfall: warning plus valid preview.
- Duplicate save: skipped and included in save statistics.

## Testing

- Parse a valid structured response.
- Reject non-JSON, missing fields, invalid options, and incompatible answers.
- Save AI drafts with `source = ai`.
- Skip duplicate AI drafts.
- Confirm missing-key page rendering does not raise.
- Confirm saved AI questions are available through existing Quiz queries.
- Preserve all V0.2 tests.

## Out of Scope

AI short-answer grading, automatic replacement calls, prompt history, usage charts,
PDF/Word/OCR, authentication, complex UI, and all V0.4 functionality.
