# StudyQuest V0.3 AI Generate Design

## Goal

Use a locally running Ollama model to generate structured question drafts,
preview them in Streamlit, and save approved drafts into the existing SQLite
question bank without duplicates or per-request API charges.

## Provider Choice

Use the official Ollama Python client and Ollama's native structured-output
support. The default model is `qwen3:8b`, selected for Chinese support and a
moderate 5.2 GB download. Users may override the model and server URL in `.env`.

Ollama runs outside StudyQuest as a local background service. StudyQuest will
not install Ollama or download models automatically. This keeps model downloads
and disk usage explicit and prevents a web-page action from starting a large
unattended download.

## Configuration

StudyQuest loads `.env` from the project root with `python-dotenv`:

```dotenv
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen3:8b
```

No API key is required. The page remains usable when Ollama is unavailable and
shows the exact setup commands instead of raising an exception.

## Components

- `modules/ai_generator.py`: configuration, Pydantic response schema, prompt
  construction, Ollama client call, normalization, and domain validation.
- `modules/importer.py`: shared duplicate-safe save function used by CSV and AI.
- `app.py`: AI form, readiness messages, session-state preview, and save action.

## Generation Flow

1. Load `.env` and render topic, type, count (1-20), and difficulty controls.
2. On Generate, call Ollama once with the `QuestionBatch` JSON schema and
   temperature `0`.
3. Validate every question against required fields and type-specific rules.
4. Normalize true/false answers to `True` or `False`, multiple-choice answers to
   a stable comma-separated label list, and non-choice options to an empty list.
5. Store validated dictionaries in `st.session_state.ai_generated_questions`.
6. Preview all required fields without writing to SQLite.
7. On Save, use the shared insert-or-ignore path with `source = ai`.
8. Show generated, inserted, duplicate, and current total counts.

If the model returns fewer valid questions than requested, keep the valid
preview and show an explicit shortfall warning. V0.3 does not make a second
generation call.

## Validation Rules

- Every item requires non-empty `subject`, `type`, `stem`, `answer`,
  `explanation`, `tags`, and `difficulty`.
- Type must equal the requested type and be one of the four supported types.
- Difficulty must equal the requested difficulty.
- Single choice requires at least two labeled options and one valid answer label.
- Multiple choice requires at least two labeled options and one or more valid,
  unique answer labels; order does not affect grading.
- True/false and short-answer questions store `options` as `[]`.
- True/false answers normalize supported English or Chinese values.
- The response root must contain `questions` as a list.

## Error Handling

- Ollama not installed or service unavailable: show installation/start guidance.
- Configured model missing: show `ollama pull <model>` guidance.
- Request timeout or transport failure: show a concise page error and preserve
  any existing preview.
- Non-JSON/schema output: report a response validation error.
- Missing or invalid domain fields: identify the invalid item.
- Count shortfall: warn and keep the valid preview.
- Duplicate save: skip it and include it in save statistics.

## Testing

- Parse a valid structured response.
- Reject non-JSON, missing fields, invalid options, and incompatible answers.
- Save AI drafts with `source = ai`.
- Skip duplicate AI drafts.
- Confirm an unavailable Ollama service does not crash the page.
- Confirm saved AI questions are available through existing Quiz queries.
- Preserve all V0.2 tests.

## Local Setup

1. Install Ollama for Windows.
2. Run `ollama pull qwen3:8b` once.
3. Ensure Ollama is running; its default API is
   `http://localhost:11434`.
4. Start StudyQuest normally. No key or paid account is required.

## Out of Scope

Automatic Ollama installation, automatic model downloads, cloud-provider
fallbacks, AI short-answer grading, replacement calls, prompt history, usage
charts, PDF/Word/OCR, authentication, complex UI, and all V0.4 functionality.
